RQ3 Report: Main Types of Adaptive Tracing and How They Differ

RQ3: What are the main types of adaptive tracing, and how do they differ in goals, decision signals, and tracing behavior?

This report answers RQ3 by synthesizing the methods implemented and evaluated in the project across all three applications. It does not introduce a new experiment matrix. Instead, it organizes the implemented controllers into a defensible taxonomy and explains how each type behaves in practice based on the completed RQ1 and RQ2 evidence.

The project evidence supports a practical five-part taxonomy. Fixed-rate tracing is the non-adaptive baseline and is reported separately from adaptive controllers when evidence is counted. Rule-based adaptive tracing reacts to explicit thresholds. Clustering-based adaptive tracing maps similar runtime states to a rate using unsupervised grouping. Bandit-style adaptive tracing learns from the immediate payoff of actions without a full transition model. Value-learning RL methods, represented by Q-Learning and SARSA, learn the long-run usefulness of tracing actions under changing runtime conditions.

Interpretation
  • Fixed rate has no runtime adaptation at all.
  • Rule can be either conservative or aggressive depending on threshold logic; in the Petclinic evidence it became the most aggressive controller under faulted conditions.
  • K-Means behaves like a moderate, stable controller and often stays near a representative middle rate.
  • Bandit tends to be conservative and low-overhead in this project, but it can still react strongly when the immediate payoff is clear.
  • SARSA is the most exploratory RL method in the pooled project evidence.
  • Q-Learning tends to stay conservative until degradation becomes clearer, then adjusts with stronger long-run reward awareness.

Showed that a simpler RL family member like Bandit can be the best choice when runtime signals are noisier and operational complexity matters.

6. Answer to RQ3

The main adaptive tracing types identified in this project are: static fixed-rate tracing, rule-based adaptive tracing, clustering-based adaptive tracing, bandit-style RL adaptive tracing, and value-learning RL adaptive tracing.

They differ along three axes:

Reward values are compared only inside the RL families. For rule-based, clustering-based, and fixed-rate methods, reward is treated as controller-specific context rather than a directly comparable cross-family score.

GoalsSome optimize low overhead, others optimize immediate payoff, and others optimize long-run reward.
Decision signalsThey range from no runtime signal at all to threshold logic, cluster assignments, and state-value updates from latency, error, and throughput.
Tracing behaviorThey differ in conservatism, stability, exploration, and willingness to change the sampling rate under degradation.

In practical terms, RQ3 shows that adaptive tracing is not one technique. It is a family of control styles. The completed experiments demonstrate that these styles are meaningfully different both in how they make decisions and in how they behave on real microservice applications.

7. Final Synthesis

The strongest project-level interpretation is that adaptive tracing should be discussed as a taxonomy of control strategies rather than as a single binary choice. Static, heuristic, clustering-based, and RL-based methods all occupy different points in the design space. The completed RQ1 and RQ2 evidence shows that:

  • RL-based value-learning methods are the most expressive and often the strongest when the system supports stable learning.
  • Bandit-style control is a useful middle ground between simple heuristics and full value-learning RL.
  • Rule-based control remains competitive when the runtime structure is simple and the system responds predictably to thresholds.
  • Clustering-based control is useful when moderate, data-driven behavior is preferred over manual thresholds.
  • Fixed-rate tracing remains an important baseline because it shows what is lost when runtime adaptation is absent.
